Important checks¶

Some checks need to be performed before starting the observations.

On nuraghe-AS (active surface desktop)¶

Check that the active surface is green (Fig.2). The active surface does not work properly if a large fraction (a whole sector) becomes red. It is a problem in K-band observations (Fig.33).

On mng-dev (observer computer)¶

Check that:
  • the log client jlog is open in order to track possible error messages (Fig.5). In case it is not open, type $ jlog & on a shell;
  • the interface of the Meteo client is open to check the wind velocity in real time (it should be < 60 km/h) (Fig.6). If the interface is closed, type $ meteoClient & on a shell.
  • all of the 33 containers are active on ACS (MNG virtual desktop) (Fig.1);

Warning

If the number of containers is 0 instead of 33 in ACS, you have to start ACS

On the nuraghe CONSOLE virtual desktop, check the presence of the 8 panels (Fig.7):
  • operatorInput (Fig.8)
  • AntennaBoss (Fig.9)
  • GenericBackend (Fig.10)
  • Mount (srt_mount_ok)
  • Observatory (Fig.14)
  • Receivers (Fig.15)
  • Scheduler (Fig.18)
  • MinorServo (Fig.20)

Upload your shedules (.scd, .lis, .bck and .cfg files) and check them:

From your computer:

$ scp  [schedulename.*] discos@mng-dev:/archive/schedules/[projectID]

On mng-dev:

$ cd /archive/schedules/[projectID]

$ scheduleChecker [schedulename.scd]
