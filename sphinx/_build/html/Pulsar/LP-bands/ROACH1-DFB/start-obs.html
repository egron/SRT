Start the observations¶

$ : commands to insert in a shell

> : commands to insert in the operatorInput panel

: check the execution on the monitor

On seadas¶

Place your observing files in pulsar@seadas as follows:

  • your seadas setup and schedule file(s) in folder: pulsar@seadas:/home/pulsar/scheds/[your project code]

On discos-console¶

  1. Insert your project number

    > project=[projectID] Fig.18

  2. Initial setup

    > antennaReset Fig.29

    > setupLLP Fig.15 Fig.27

    > goTo=*,81.9d Fig.12

  3. On a terminal open a vnc session for pulsar@seadas:

    $ vncviewer seadas:1

  4. If this fails, start a new vnc session on pulsar@seadas.

  5. Type password at prompt.

  6. On another workspace, open a new terminal and start a vnc session for corr@psrdfb:

    $ vncviewer psrdfb:2

  7. If this fails, start a new vnc session on corr@psrdfb.

  8. Type password at prompt.

  9. On another workspace, open a new terminal and start a vnc session for the head node leap0:

    $ vncviewer leap0:1

  10. If it is not active log on to the LEAP cluster :

    $ ssh -X user@leap0 ask for password

  11. Launch a VNC session once logged on to leap0 :

    leap0: $ vncserver &

In the vnc session pulsar@seadas¶

  1. Open a terminal and start SEADAS

    $ seadas

  2. In the log frame of seadas main window check that SEADAS connects to nuraghe

  3. Click on the red label at the top right corner of the Antenna and Pointing Management frame.

  4. Check that the clicked label becomes green and dislpays the word “ENABLED”

In the vnc session corr@psrdfb¶

  1. Open a terminal and check the system clock is working properly (time properly synchronized, tick phase ~ 100 ns, positive or negative) by typing

    $ atdc

  2. Open a new terminal and start DFBCONTROLLER

    $ dfbcontroller

  3. In DFBCONTROLLER window, check that the colored label at the right of the

“tkds” label is green and displays the word “CONNECTED”. If not, click on it.

In the vnc session leap0¶

  1. If you do not see the LEAPCONTROLLER interface, open a new terminal and start it

    $ /home/user/seadas/bin/leapcontroller

  2. In LEAPCONTROLLER window, check that the colored label at the right of the “…” label is green and displays the word “CONNECTED”. If not, click on it.

Inside SEADAS window¶

  1. Select “Schedule” in the “Session mode” combo box
  2. Verify and adjust attenuation levels. You can do so by following the procedure below, using the schedule TEST.scd which you will find under the directory /home/corr/scheds/. Once you have started the schedule (see points below) you need to check the PDFB3 panel in the psrdfb vnc session. Under tab … make sure that the RMS level for each channel is around 10. If it is not, go back to the SEADAS interface and select different attenuation values (which can be different for the two channels) and press set atten.
  3. Click “Schedule Management”. A window named “Seadas Schedule Manager” pops up
  4. Click “Load sched” in “Schedule Manager” window. A system window pops up for browsing system directories and selecting the schedule file
  5. In “Schedule manager” window select schedule lines to be done - mouse left click on each single line - or click button “Select all” for loading the entire schedule in the “Observations List” window
  6. If necessary, rearrange the order of the observations in the “Observations List” window - left button click == cut line ; mid button click == paste line
  7. Click “Observe”
